Links (Hard & Soft)::
=====================

ln file hardlink â†’ Creates a hard link pointing to the same inode as the original file

ğŸ”¹Hard Link 

Uses the same inode number as the original file
Changes made via any link affect the same data
Cannot be created for directories (normal users)
Cannot span across different filesystems
File exists until all hard links are removed

ğŸ”¹Soft Link 

Has a different inode number
Points to the file path, not the inode
Can link files and directories
Can cross filesystems
Becomes broken if the original file is deleted.

Linux File System Hierarchy
==============================
/  (Root Directory)
------------------
- Top-level directory
- Parent of all directories
- Represented by /
- Similar to C:\ in Windows
Commands:
ls /
cd /
pwd

/root
-----
- Home directory of root (super user)
- Not accessible by normal users
Commands:
sudo ls /root
sudo -i

/home
-----
- Home directory for all normal users
- Each user has a separate folder
- Example: /home/alex
Commands:
ls /home
cd /home/username
whoami

/bin(Binary Files)
-------------------
- Essential commands for all users
- Required in single-user mode

LVM PARTITION:
LVM (Logical Volume Manager) allows flexible disk management by abstracting physical storage.

Physical Volume (PV) â†’ A disk or partition initialized for LVM
Volume Group (VG) â†’ A pool of storage created from one or more PVs
Logical Volume (LV) â†’ Virtual partition created from VG space

Basic flow:
Disk/Partition â†’ PV â†’ VG â†’ LV â†’ Filesystem â†’ Mount point

LVM Creation::
lsblk â†’ Lists all disks and partitions to identify the correct target disk

sudo parted /dev/nvme1n1 print â†’ Displays existing partition layout as a dry-run

sudo parted /dev/nvme1n1 --align optimal â†’ Opens parted with optimal alignment for SSD/NVMe

mklabel gpt â†’ Creates a GPT partition table (destroys existing layout)

mkpart primary 1MiB 100% â†’ Creates a single partition for LVM use

set 1 lvm on â†’ Marks the partition type as LVM

print â†’ Displays the partition table for confirmation

quit â†’ Exits parted after verification

lsblk /dev/nvme1n1 â†’ Confirms the kernel detects the new partition

pvcreate /dev/nvme1n1p1 â†’ Initializes the partition as an LVM Physical Volume

vgcreate vg_data /dev/nvme1n1p1 â†’ Creates a Volume Group named vg_data

vgdisplay â†’ Displays Volume Group details

mkfs.xfs /dev/vg_data/lv_data â†’ Formats the Logical Volume with XFS

mkdir -p /data â†’ Creates a mount point

umount /data && mount -a â†’ Tests permanent mount without reboot.
